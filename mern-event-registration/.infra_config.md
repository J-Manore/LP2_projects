# 🎟️ Event Registration: Ultimate VM Deployment Guide

This guide provides a **foolproof, step-by-step process** to deploy an Event Registration system on a clean Ubuntu Linux VM (Azure/AWS). No managed services are used; everything runs locally on your VM.

> [!TIP]
> **Viva Flex:** "This deployment uses a self-hosted MongoDB instance on the VM instead of managed cloud databases, ensuring full control over infrastructure and avoiding vendor lock-in."

## 1. Project Overview
A CRUD application for managing event registrations (Event Name, Organizer, Date, Location, Description). Features a React frontend and a Node.js/Express backend connected to a local MongoDB instance.

## 2. Tech Stack
- **Frontend:** React.js (Vite), Axios
- **Backend:** Node.js, Express.js
- **Database:** MongoDB (Local Instance)
- **Process Manager:** PM2

---

## 3. Azure VM Configuration (Portal)
**CRITICAL:** You must open the ports in the Azure Portal before the app can be accessed.

1. Navigate to your **VM** > **Networking**.
2. Click **Add inbound port rule**.
3. Create rules for:
   - **Port:** `5000` (Backend)
   - **Port:** `3000` (Frontend)
4. Note your **Public IP Address**.

## 4. Server Setup (Terminal/SSH)

### A. Connect to VM
```bash
ssh username@your-vm-public-ip
```

### B. Update & Install Basic Tools
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl build-essential

# Configure Local Firewall (UFW)
sudo ufw allow 3000
sudo ufw allow 5000
sudo ufw allow 22
sudo ufw --force enable
```

### C. Install Node.js (Version 18.x)
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
# Verify Installation
node -v
npm -v
```

### D. Install MongoDB (Official Community Edition)
```bash
# Import GPG Key
curl -fsSL https://www.mongodb.org/static/pgp/server-6.0.asc | sudo gpg --dearmor -o /usr/share/keyrings/mongodb-server-6.0.gpg

# Add Repo (Official 6.0 Script)
echo "deb [arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-6.0.gpg] https://repo.mongodb.org/apt/ubuntu $(lsb_release -sc)/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list

# Install
sudo apt update
sudo apt install -y mongodb-org

# Start & Enable
sudo systemctl start mongod
sudo systemctl enable mongod

# Verify Service & CLI
sudo systemctl status mongod
mongosh --eval "db.adminCommand('ping')"
```

## 5. Application Deployment

### A. Clone and Setup Backend
```bash
git clone <your-repo-url>
cd mern-event-registration/backend
npm install
# Create .env
cat <<EOT > .env
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/event-db
EOT

# Test Backend Locally (CRITICAL CHECK)
curl http://localhost:5000/api/events
```

### B. Frontend Setup (The "IP Switch" Step)
**IMPORTANT:** Build the frontend with the VM's Public IP.

```bash
cd ../frontend
npm install
# Build with environment variable (Replace YOUR_VM_IP)
VITE_API_URL=http://YOUR_VM_IP:5000/api/events npm run build
```

## 6. Process Management (PM2)
```bash
sudo npm install -g pm2
sudo npm install -g serve

# Start Backend
cd ../backend
pm2 start server.js --name "event-api"

# Start Frontend
cd ../frontend
pm2 serve dist 3000 --name "event-ui" --spa

# Save for persistence
pm2 save
pm2 startup
```

## 7. Debugging & Troubleshooting

| Issue | Solution | Command |
| :--- | :--- | :--- |
| **App not loading** | Check Azure NSG rules | Azure Portal UI |
| **MongoDB Connection Error** | Restart Mongo / Use `127.0.0.1` | `sudo systemctl restart mongod` |
| **Version Mismatch** | Ensure Node.js 18+ is installed | `node -v` |
| **Frontend can't reach API** | Rebuild with correct Public IP | See Step 5B |
| **CORS Errors** | Ensure CORS is enabled | `app.use(require('cors')())` |
| **Mongo Bind Error** | Check `/etc/mongod.conf` | Ensure `bindIp: 127.0.0.1` |
| **Application Crash** | Check PM2 logs | `pm2 logs` |

### Viva Tip: "Why didn't you use localhost in the frontend build?"
*Answer:* React runs in the user's browser. If we use `localhost`, it looks for the API on the *user's* computer. We must use the server's Public IP to connect correctly.
