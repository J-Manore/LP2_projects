# 🔒 Secure File Sharing (FINAL EXAM VERSION)

## 🧠 FLOW
```
VM-A → SSH setup → connect to VM-B → transfer file → verify
```

---

# ⚙️ 1. PREREQUISITES
* Two VMs in same VNet
* Both running
* Port 22 (SSH) open

## 📌 Get Private IPs
```bash
hostname -I
```
*Example:*
* VM-A: `10.0.0.4`
* VM-B: `10.0.0.5`

---

# 🔑 2. SETUP PASSWORDLESS SSH (BEST METHOD)
**On VM-A:**
```bash
ssh-keygen -t rsa -b 4096
```
👉 *Press Enter for everything.*

**Copy key to VM-B:**
```bash
ssh-copy-id azureuser@10.0.0.5
```
👉 *Now VM-A can access VM-B without password*

---

# 📤 3. FILE TRANSFER

## ✅ Push file (A → B)
```bash
scp hello.py azureuser@10.0.0.5:~
```

## ✅ Pull file (B → A)
```bash
scp azureuser@10.0.0.5:~/hello.py .
```

---

# 🔍 4. VERIFY
**On VM-B:**
```bash
ls ~
```

**Optional (execute remotely):**
```bash
ssh azureuser@10.0.0.5 'ls ~'
```

---

# 🔐 5. SECURITY (OPTIONAL BUT GOOD FOR VIVA)
```bash
mkdir secure_folder
chmod 700 secure_folder
mv hello.py secure_folder/
```

---

# 🐛 DEBUGGING

❌ **Permission denied**
👉 `ssh-copy-id azureuser@10.0.0.5`

❌ **Connection timeout**
👉 `ping 10.0.0.5`

❌ **Wrong IP**
👉 `hostname -I`

❌ **SSH not working**
👉 Check Azure port 22

---

# 🧠 VIVA ANSWERS

**❓ Why SCP?**
👉 Uses SSH → encrypted file transfer

**❓ Why private IP?**
👉 stays inside VNet → secure + faster

**❓ SCP vs FTP?**
👉 SCP = encrypted
👉 FTP = plain text (unsafe)
