# 🔒 Secure File Sharing: Ultimate VM Deployment Guide

This project demonstrates how to transfer files securely between two Cloud VMs using the terminal. It focuses on SSH key management and secure protocols.

> [!TIP]
> **Viva Flex:** "We use SCP (Secure Copy Protocol) because it runs over SSH, ensuring that the file is encrypted during transmission. By using Private IPs, we keep the traffic within our Virtual Network, which is more secure and avoids data transfer costs over the public internet."

---

## 1. Prerequisites & Azure Portal Setup
1. **Create Two VMs:** Ensure both are in the same Resource Group and **Virtual Network (VNet)**.
2. **Networking (Portal):** 
   - Navigate to **Networking** for both VMs.
   - Ensure **Port 22 (SSH)** is open in the Inbound Port Rules.
3. **Note IPs:**
   - VM-A (Sender): `10.0.0.4`
   - VM-B (Receiver): `10.0.0.5`

---

## 2. Setup Connectivity (VM-A to VM-B)

### A. Generate SSH Key on VM-A
Generate a cryptographic identity to allow passwordless access.
```bash
ssh-keygen -t rsa -b 4096
# Press Enter for all prompts (no passphrase for practicals)
```

### B. Propagate Key to VM-B
Authorize VM-A to access VM-B.
```bash
ssh-copy-id azureuser@10.0.0.5
```
*Note: You will be asked for VM-B's password once.*

---

## 3. Secure File Transfer (Terminal)

### A. Push File (A ➡️ B)
From the VM-A terminal, push the Python file:
```bash
scp hello.py azureuser@10.0.0.5:~/
```

### B. Pull File (A ⬅️ B)
Pull a file from VM-B to the current directory on VM-A:
```bash
scp azureuser@10.0.0.5:~/hello.py ./hello_received.py
```

---

## 4. Verification & Folder Hardening

### A. Execute Remotely
```bash
ssh azureuser@10.0.0.5 'python3 ~/hello.py'
```

### B. POSIX Permission Hardening
Secure a folder so only the owner can read/write:
```bash
mkdir secure_vault
chmod 700 secure_vault
mv hello.py secure_vault/
```

---

## 5. Extensive Troubleshooting

| Issue | Solution | Command/Check |
| :--- | :--- | :--- |
| **Connection Timeout** | Check if VM-B is running and Port 22 is open in Azure. | `ping 10.0.0.5` |
| **Permission Denied (publickey)** | Key wasn't copied correctly. | `ssh-copy-id azureuser@10.0.0.5` |
| **Host Key Changed** | Someone reinstalled the OS on VM-B. | `ssh-keygen -R 10.0.0.5` |
| **Command not found** | Ensure OpenSSH is installed. | `sudo apt install openssh-client` |
| **Wrong Private IP** | Verify your internal IP. | `hostname -I` |

### Viva Tip: "What is the difference between SCP and FTP?"
*Answer:* SCP is built on top of SSH and is inherently secure and encrypted. FTP is an older protocol that sends data (including passwords) in plain text unless wrapped in SSL (FTPS). For cloud management, SCP/SFTP is the industry standard.
