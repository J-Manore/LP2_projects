# 🌐 Static Website Deployment (FINAL SIMPLE VERSION)

## 🧠 FLOW
```
SSH → install nginx → upload files → move → restart → open browser
```

---

# 🔑 1. SSH
```bash
ssh -i ~/key.pem azureuser@<VM-IP>
```

---

# ⚙️ 2. INSTALL NGINX
```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

---

# 🔥 3. OPEN PORT (VERY IMPORTANT)
👉 **Azure Portal:**
Add inbound rule → Port 80

---

# 🧪 4. TEST DEFAULT PAGE
Open:
`http://<VM-IP>`

👉 If you see “Welcome to nginx” → good

---

# 📤 5. UPLOAD FILES (FROM LOCAL MACHINE)
```bash
scp -i ~/key.pem index.html azureuser@<VM-IP>:~
```

👉 If multiple files:
```bash
scp -i ~/key.pem -r website/ azureuser@<VM-IP>:~
```

---

# 📁 6. MOVE FILES (INSIDE VM)
```bash
sudo rm -rf /var/www/html/*

# For single file:
sudo mv ~/index.html /var/www/html/

# For folder:
sudo mv ~/website/* /var/www/html/
```

---

# 🔁 7. RESTART NGINX
```bash
sudo nginx -t
sudo systemctl reload nginx
```

---

# 🌐 8. OPEN WEBSITE
`http://<VM-IP>`

---

# 🐛 DEBUGGING (IMPORTANT)

## 1. Site not loading
👉 Check port 80 open in Azure

## 2. 403 Forbidden
```bash
sudo chmod -R 755 /var/www/html
```

## 3. Still default nginx page
```bash
sudo rm /var/www/html/index.nginx-debian.html
```

## 4. File not updating
👉 Clear browser cache or: `Ctrl + Shift + R`

## 5. Nginx not running
```bash
sudo systemctl status nginx
```

## 6. Port conflict
```bash
sudo lsof -i :80
```

# 🧠 VIVA GOLD ANSWERS

**❓ Why Nginx?**
👉 “Nginx is event-driven and lightweight, so it handles multiple requests efficiently and is ideal for static content.”

**❓ What does /var/www/html do?**
👉 “It is the default root directory where Nginx serves static files.”

**❓ What is SCP?**
👉 “Secure Copy Protocol used to transfer files securely from local machine to server.”
