# 🌐 Static Website Deployment: Ultimate VM Deployment Guide

Deploy a premium static website to a Cloud Instance using raw terminal commands and Nginx. This guide focuses on manual configuration and security hardening.

> [!TIP]
> **Viva Flex:** "This project demonstrates the transition from local development to production hosting. By using Nginx on a Linux VM, we achieve higher performance and better security control compared to simple shared hosting."

---

## 1. Azure Portal Configuration
1. Navigate to your **VM** > **Networking**.
2. Click **Add inbound port rule**.
3. Create a rule for:
   - **Protocol:** TCP
   - **Port:** `80` (HTTP)
   - **Priority:** 310 (or lower than deny rules)
4. Note your **Public IP Address**.

---

## 2. Server Setup (Terminal/SSH)

### A. Connect to VM
```bash
ssh username@your-public-ip
```

### B. Install & Enable Nginx
```bash
sudo apt update
sudo apt install -y nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

### C. Firewall Configuration (UFW)
```bash
sudo ufw allow 80/tcp
sudo ufw allow 22/tcp
sudo ufw --force enable
```

---

## 3. Application Deployment

### A. Upload Files (From Local Machine)
Run this command from your project root on your **local computer**:
```bash
scp -r website/* username@your-public-ip:~/
```

### B. Configure Web Root (On VM)
Move the uploaded files to the Nginx default serving directory.
```bash
# Clear default Nginx page
sudo rm -rf /var/www/html/*

# Move new files
sudo mv ~/index.html /var/www/html/
# (If you have CSS/JS folders, move those too)
sudo mv ~/css /var/www/html/ 2>/dev/null
```

### C. Verify Deployment
```bash
sudo nginx -t
sudo systemctl reload nginx
```

---

## 4. Advanced: Custom Nginx Config (Optional)
To point Nginx to a specific folder:
```bash
sudo nano /etc/nginx/sites-available/default
# Change 'root /var/www/html;' to your preferred path
```

## 5. Extensive Troubleshooting

| Issue | Solution | Command/Check |
| :--- | :--- | :--- |
| **Welcome to Nginx page still shows** | Clear browser cache or remove `index.nginx-debian.html`. | `sudo rm /var/www/html/*.html` |
| **403 Forbidden** | Nginx doesn't have permission to read the files. | `sudo chmod -R 755 /var/www/html` |
| **Connection Timed Out** | Azure NSG rule for Port 80 is missing or UFW is blocking it. | `sudo ufw status` |
| **Nginx Syntax Error** | Typo in configuration files. | `sudo nginx -t` |
| **Port 80 in use** | Another service (like Apache) is running on Port 80. | `sudo lsof -i :80` |

### Viva Tip: "Why use Nginx over Apache for static sites?"
*Answer:* Nginx uses an asynchronous, event-driven architecture which allows it to handle thousands of concurrent connections with very low memory overhead, making it significantly faster for serving static assets like HTML, CSS, and Images.
