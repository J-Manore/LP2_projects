## 1. Prerequisites
- **Azure Account:** A subscription to create a VM.
- **SSH Client:** Terminal (Linux/Mac) or PowerShell/PuTTY (Windows).
- **VM Specs:** Ubuntu 20.04 or 22.04 LTS (Standard B1s or B2s is fine).

---

## 1. Project Overview
A CRUD application for blog posts with a React frontend and Node.js/Express backend. It uses a local MongoDB instance running on the same server.

> [!TIP]
> **Viva Flex:** "This deployment uses a self-hosted MongoDB instance on the VM instead of managed cloud databases, ensuring full control over infrastructure and avoiding vendor lock-in."

## 2. Local Development Setup
Before deploying, ensure it works on your machine:
1. `npm install` in both `frontend` and `backend` folders.
2. Start MongoDB locally.
3. Run `npm start` (backend) and `npm run dev` (frontend).

---

## 3. Azure VM Configuration (Portal)
**CRITICAL STEP:** Before you even touch the terminal, you must open the ports in the Azure Portal.

1. Navigate to your **Virtual Machine** in the Azure Portal.
2. Go to **Networking** (on the left sidebar).
3. Click **Add inbound port rule**.
4. Create two rules:
   - **Rule 1:** Port: `5000` | Protocol: `TCP` | Name: `AllowBackend`
   - **Rule 2:** Port: `3000` | Protocol: `TCP` | Name: `AllowFrontend`
5. Note your **Public IP Address** (e.g., `20.12.34.56`).

## 4. Server Setup (Terminal/SSH)

### A. Connect to VM
```bash
ssh username@your-vm-public-ip
```

### B. Update & Install Basic Tools
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y git curl build-essential

# Configure Firewall (VM side)
sudo ufw allow 3000
sudo ufw allow 5000
sudo ufw allow 22
sudo ufw --force enable
```

### C. Install Node.js (Version 18.x)
```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
# Verify Installation
node -v
npm -v
```

### D. Install MongoDB (Official Community Edition)
```bash
# Import GPG Key
curl -fsSL https://www.mongodb.org/static/pgp/server-6.0.asc | sudo gpg --dearmor -o /usr/share/keyrings/mongodb-server-6.0.gpg

# Add Repo
echo "deb [arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-6.0.gpg] https://repo.mongodb.org/apt/ubuntu $(lsb_release -sc)/mongodb-org/6.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-6.0.list

# Install
sudo apt update
sudo apt install -y mongodb-org

# Start & Enable
sudo systemctl start mongod
sudo systemctl enable mongod

# Verify Service
sudo systemctl status mongod

# Verify MongoDB (CLI)
mongosh --eval "db.adminCommand('ping')"
# Or simply run: mongosh
# Then type: show dbs
```

## 5. Application Deployment

### A. Clone the Project
```bash
git clone <your-github-repo-url>
cd mern-blog-app
```

### B. Backend Setup
```bash
cd backend
npm install
# Create .env
cat <<EOT > .env
PORT=5000
MONGO_URI=mongodb://127.0.0.1:27017/mern-blog
EOT

# Test Backend Locally (CRITICAL CHECK)
# If this returns JSON, your backend is ready!
curl http://localhost:5000/api/blogs
```

### C. Frontend Setup (The "IP Switch" Step)
**IMPORTANT:** You must build the frontend with the VM's Public IP so the browser knows where the API is. Replace `YOUR_VM_IP` below.

```bash
cd ../frontend
npm install
# Build with environment variable
VITE_API_URL=http://YOUR_VM_IP:5000/api/blogs npm run build
```

## 6. Process Management (PM2)
We use PM2 to keep the apps running even if you close the terminal.

### A. Install PM2
```bash
sudo npm install -g pm2
```

### B. Start Backend
```bash
cd ../backend
pm2 start server.js --name "api"
```

### C. Start Frontend
We will use a simple static server to serve the React `dist` folder.
```bash
sudo npm install -g serve
pm2 serve ../frontend/dist 3000 --name "ui" --spa
```

### D. Verify & Save
```bash
pm2 status
pm2 save
# Optional: Setup startup script
pm2 startup
```

## 7. Debugging & Troubleshooting

| Issue | Solution | Command |
| :--- | :--- | :--- |
| **App not loading** | Check if ports are open in Azure NSG | Azure Portal UI |
| **MongoDB Connection Error** | Restart MongoDB service / Use `127.0.0.1` | `sudo systemctl restart mongod` |
| **Version Mismatch** | Ensure Node.js 18+ is installed | `node -v` |
| **Frontend can't reach API** | Rebuild frontend with correct `VITE_API_URL` | See Step 5C |
| **Port Conflict** | Check what is running on a port | `sudo lsof -i :5000` |
| **Application Crash** | View real-time logs | `pm2 logs` |
| **CORS Errors** | Ensure CORS is enabled in backend | `app.use(require('cors')())` |
| **Mongo Bind Error** | Check bindIp in config | See below |

### Fixing MongoDB Bind IP (If connection fails)
If MongoDB doesn't accept local connections, check the config:
1. `sudo nano /etc/mongod.conf`
2. Ensure the network interface section looks like this:
   ```yaml
   net:
     port: 27017
     bindIp: 127.0.0.1
   ```
3. Restart Mongo: `sudo systemctl restart mongod`

### Viva Tip: "Why didn't you use localhost in the frontend build?"
*Answer:* Because the React app runs in the **client's browser**. If we use `localhost`, the browser will try to find the API on the user's computer, not our server. We must use the VM's Public IP.
